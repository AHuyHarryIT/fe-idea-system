# FE Idea System

Role-based frontend for a University Idea Collection System.

Built with React, TypeScript, TanStack Router, TanStack Query, Vite, and Tailwind CSS.

## Features

- JWT-based authentication flow
- Role-based protected routing (`staff`, `qa_coordinator`, `qa_manager`, `admin`)
- Idea listing, filtering, detail, voting, and commenting
- Idea submission with multipart file upload
- Dashboard views for each role
- Admin overview (users, categories, submissions, ideas)
- CSV/ZIP export service support

## Tech Stack

- React 19 + TypeScript
- Vite 7
- TanStack Router (file-based routing)
- TanStack Query
- Tailwind CSS 4
- Lucide React icons

## Requirements

- Node.js 18+ (recommended 20+)
- npm

## Environment Variables

Create a `.env` file in the project root:

```env
VITE_API_URL=http://localhost:5000/api
```
## Getting Started

Install dependencies:

```bash
npm install
```

Run development server:

```bash
npm run dev
```

The app starts on http://localhost:5173.

## Scripts

## Application Routing

Main routes:

- `/login` - public login page
- `/` - redirects to role-specific home
- `/dashboard` - staff dashboard
- `/submit-idea` - staff submit idea page
- `/ideas` - shared idea listing
- `/ideas/$ideaId` - idea detail page
- `/qa-coordinator` - QA Coordinator dashboard
- `/qa-manager` - QA Manager dashboard
- `/admin` - Admin dashboard

Route protection is handled in `ProtectedPage` with role checks and redirects.

## Roles & Access

- `staff`: dashboard, submit idea, browse ideas, vote/comment
- `qa_coordinator`: coordinator dashboard + browse ideas
- `qa_manager`: manager dashboard + browse ideas
- `admin`: admin dashboard + browse ideas

Invalid or missing auth redirects to `/login`.

## API Integration

Base URL is read from `VITE_API_URL`.

Core API modules:

- `authService` - login/register
- `ideaService` - idea CRUD-like read/submit, vote, comment
- `categoryService` - category reads and management endpoints
- `submissionService` - submission periods
- `dashboardService` - dashboard/statistics endpoints
- `userService` - admin user management
- `exportService` - CSV/ZIP download endpoints

The app uses a shared `ApiClient` wrapper for:

- auth header injection
- JSON/text response parsing
- normalized `ApiResponse<T>` output
- auto-logout on `401`

## Data Mapping Strategy

`src/lib/api-mappers.ts` normalizes backend payload variations into stable frontend models:

- `mapIdeaSummary`
- `mapIdeaDetail`
- `mapCategory`
- `mapSubmission`

This allows the UI to remain stable even when backend field shapes vary.

## Project Structure

```text
src/
  api/               # service modules + shared API client
  components/        # reusable UI components
  constants/         # navigation by role
  hooks/             # TanStack Query hooks
  integrations/      # query provider + devtools integration
  Layout/            # shell layout (top nav + sidebar)
  lib/               # auth and API data mapping utilities
  pages/             # route page implementations by role
  routes/            # file-based route definitions
  types/             # shared app types
```

## Notes

- `src/routeTree.gen.ts` is auto-generated by TanStack Router. Do not edit manually.
- Tokens and roles are persisted in `localStorage`.
- Export endpoints download files using browser blob URLs.
